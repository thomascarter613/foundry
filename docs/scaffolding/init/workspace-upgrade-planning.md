# Workspace Upgrade Planning

The generated workspace upgrade planner is the first step toward post-v1 workspace migration support.

## Purpose

Generated workspaces may need future upgrades as `foundry init` evolves.

The planner detects whether a workspace was generated by `foundry init`, compares it against the current v1 baseline, and returns a read-only upgrade plan.

## Current scope

The current planner is read-only.

It does not write files.

It does not modify provenance.

It does not mutate generated workspaces.

## Detection

The planner checks for:

```text
.foundry/
.foundry/init/provenance.json
.foundry/init/audit.ndjson
config/foundry/generator-manifest.json
A workspace is considered generated by foundry init when provenance indicates:

￼
generatedBy.command = "foundry init"
Plan statuses
The planner can return:

Status	Meaning
not-foundry-workspace	No Foundry init provenance was detected.
invalid-foundry-workspace	Foundry metadata exists but does not prove foundry init generated it.
current	Required v1 baseline files are present.
upgrade-available	One or more required v1 baseline files are missing.
￼
Required v1 files
The planner checks for the generated workspace baseline, including:

￼
package.json
README.md
bunfig.toml
.gitignore
.gitattributes
.editorconfig
CONTRIBUTING.md
SECURITY.md
.github/workflows/ci.yml
.github/pull_request_template.md
tools/scripts/foundry.sh
tools/scripts/verify.sh
packages/cli/src/index.ts
config/foundry/generator-manifest.json
.scaffdog/config.js
docs/ai/BOOTSTRAP_PROMPT.md
docs/ai/CURRENT_STATE.md
.foundry/README.md
.foundry/init/provenance.json
.foundry/init/audit.ndjson
Future mutation command
A later foundry init upgrade command should consume this planner.

The mutating command should:

print the plan first;

require explicit execution;

avoid overwriting user-edited files without confirmation or backup;

append audit events;

preserve existing provenance;

produce deterministic output.

Policy
Upgrade planning must remain safe by default.

Read-only planning should be available before mutating upgrades.

## Command

The read-only upgrade command is:

```bash
bun run foundry -- init upgrade
To inspect a specific workspace:

Bash
￼
bun run foundry -- init upgrade myapp
To print JSON:

Bash
￼
bun run foundry -- init upgrade myapp --json
To use the command as a CI check:

Bash
￼
bun run foundry -- init upgrade myapp --check
--check exits successfully only when the workspace status is current.

Mutation policy
The current command is read-only.

It does not create, edit, remove, or overwrite files.

A future mutating upgrade command must require explicit execution and must preserve or audit user-edited files.
