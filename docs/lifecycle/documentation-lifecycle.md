# Documentation Lifecycle

## Purpose

Define how documentation work moves from idea to governed artifact.

## Workflow

1. Propose documentation change.
2. Generate or edit governed artifact.
3. Validate metadata, structure, links, and graph impact.
4. Review according to governance level.
5. Merge only after verification passes.

## Roles & Responsibilities

- Owner: Maintains document accuracy.
- Reviewer: Validates content and links.
- Governance: Resolves binding rule changes.

## Required Artifacts

- Governed Markdown document.
- Valid metadata.
- Upstream/downstream links.
- Governance links.
- ADR links where applicable.

## Approval Gates

- Informational: reviewer approval.
- Required: domain owner approval.
- Binding: governance approval.

## Escalation Paths

Escalate unresolved conflicts to the document owner and governance authority.
